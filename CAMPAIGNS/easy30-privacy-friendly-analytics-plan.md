# Easy30 Privacy-Friendly Analytics Plan

Last updated: 2026-05-29

## Purpose

Track basic site performance and traffic without adding heavy tracking scripts to ReadEasy30 or MathEasy30.

## Default analytics approach

Use Cloudflare dashboard analytics first.

This avoids adding JavaScript tracking snippets to the website code.

## Why this fits Easy30

- no extra script weight
- no tracking pop-ups added to lesson pages
- no extra JavaScript dependency
- no learner account required
- simple traffic visibility from the hosting layer

## Cloudflare steps

1. Log in to Cloudflare.
2. Open the domain, such as `readeasy30.com` or `matheasy30.com`.
3. Open Analytics & Logs.
4. Open Traffic.
5. Review daily visitors, pageviews, geography, and performance indicators.

## Metrics to watch

- unique visitors
- pageviews
- top pages if available
- traffic by country or region
- cache performance
- site speed indicators
- 404 or error trends if available

## Outreach use

After posting to Pinterest, Facebook, Reddit, or library contacts, check Cloudflare traffic within 24 to 72 hours.

Record results in:

`CAMPAIGNS/easy30-outreach-tracking-template.csv`

## Privacy wording rule

Do not claim “zero tracking” publicly unless all site behavior is verified.

Safer wording:

- no account required
- no tracking scripts added for launch analytics
- basic traffic is reviewed through Cloudflare hosting analytics
- no learner profile is required to practice

## Do not add without approval

- Google Analytics
- Meta Pixel
- TikTok Pixel
- ad tracking tags
- heatmap tools
- session recording tools
- live remarketing scripts

## Weekly review routine

Every week, record:

1. Which post or channel likely created traffic?
2. Which page got the most visits?
3. Which country or region showed interest?
4. Which outreach contact replied?
5. Which page should be improved next?

## Decision rule

If Cloudflare analytics gives enough information, do not add external tracking.

Keep the learning experience fast, calm, and privacy-friendly.
