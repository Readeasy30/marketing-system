# Daily Education Marketing Operating System

Last updated: 2026-06-05

Campaign family: **ReadEasy30 + MathEasy30**

Purpose: Run education marketing every day without restarting, rebuilding, or hunting through old files.

## Plain rule

Marketing is now a daily operating system, not a one-time launch push.

Every day should end with one of these outcomes:

1. A post is published.
2. A blocker is recorded.
3. A small safe improvement is completed.
4. The next exact action is locked for tomorrow.

Do not create new marketing assets just because posting feels blocked.

## Daily decision tree

### Step 1 — Is one social account ready?

Check:

```text
EDUCATION-BATCH-01-ACCOUNT-READINESS-BOARD.md
```

If one account is ready, post the next asset.

Best first order:

1. Facebook Page or LinkedIn
2. YouTube Shorts / Reels
3. Pinterest
4. TikTok
5. X later

### Step 2 — If no account is ready

Do not make more content.

Record the blocker and do one safe setup action:

- choose the first account to open
- verify profile name
- verify profile bio
- verify website link
- prepare the post text
- confirm the correct image/video file

### Step 3 — If posting is done

Update both tracking files:

```text
EDUCATION-BATCH-01-POSTING-LOG.md
CAMPAIGN-PUBLISHING-TRACKER.md
```

Then move to the next asset in the Batch 01 queue.

### Step 4 — If posting stays blocked

Use the day for safe marketing-system cleanup only:

- simplify instructions
- update status files
- check live links
- fix typos in marketing files
- prepare tomorrow's exact one-action instruction

Do not edit live product websites unless the task is a safe marketing marker, typo fix, broken-link fix, sitemap/robots/status update, or social preview fix.

## Daily work modes

### Mode A — Posting mode

Use when a social account is ready.

Goal: publish one post and record the link.

Files to use:

```text
EDUCATION-BATCH-01-FIRST-POST-CLIPBOARD.md
EDUCATION-BATCH-01-FIRST-VIDEO-CLIPBOARD.md
EDUCATION-BATCH-01-WORKSHEET-PINS-CLIPBOARD.md
EDUCATION-BATCH-01-ADULT-LEARNER-CLIPBOARD.md
EDUCATION-BATCH-01-OPTIONAL-VIDEOS-CLIPBOARD.md
```

### Mode B — Blocker mode

Use when Gerry does not have the account open or ready.

Goal: record the blocker clearly and choose the next account to try.

Example blocker:

```md
| 2026-06-05 | Facebook Page | Card 1 launch post | Blocked | Page login not ready | Try LinkedIn next |
```

### Mode C — Safe improvement mode

Use only after the daily posting/blocker status is clear.

Allowed:

- link checks
- typo fixes
- status updates
- sitemap/robots notes
- social preview notes
- simple marketing file cleanup

Not allowed:

- redesigns
- framework changes
- paid ads
- tracking scripts
- affiliate links
- payment tools
- API keys
- scraping
- automated posting
- account systems

### Mode D — New content mode

Use only after Batch 01 is posted, blocked, paused, or intentionally moved forward.

Until then, new content is lower priority than getting the first public post live.

## What ChatGPT should do each day

1. Find the current locked campaign.
2. Read the account readiness board.
3. Read the posting log.
4. Tell Gerry the one next action.
5. Provide copy/paste text only for that action.
6. After Gerry provides a live link or blocker, update the posting log and campaign tracker.
7. Keep the product repos stable unless a safe marketing fix is needed.

## What Gerry must do manually

Gerry must manually log in and post on external social platforms.

ChatGPT should not:

- ask for passwords
- store passwords
- automate posting
- scrape contacts
- create fake accounts
- bypass platform controls
