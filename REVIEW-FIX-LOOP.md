# Review Fix Loop

## Purpose

Use this file to guide AI and Codex through better website production.

## Main rule

AI and Codex do the file work directly.

Do not send file creation, paste work, or file updates back to Gerry.

## Review order

For each active website repo:

1. Read README.
2. Read AGENTS.md.
3. Read AGENT-INSTRUCTIONS.md when present.
4. Check main HTML files.
5. Check main CSS files.
6. Check main JavaScript files.
7. Check support pages.
8. Fix safe issues directly.
9. Commit useful changes.
10. Record blockers.

## Common issues to fix

- broken links
- wrong CSS paths
- duplicate style files
- missing AGENTS.md
- missing README instructions
- old starter app files
- confusing file names
- weak mobile layout
- unclear homepage message
- missing support pages
- repeated planning files

## Safe fixes

- update HTML
- update CSS
- update JavaScript
- update Markdown
- improve copy
- improve navigation
- improve app controls
- improve mobile layout
- add simple missing support files
- remove duplicate files after checking references

## Stop points

Stop only for:

- blocked writes
- major code deletion
- framework changes
- private keys
- payment setup
- live tracking
- live ads
- unclear repo state

## Report format

After a review batch, report:

1. Repos checked
2. Files changed
3. Issues fixed
4. Blocked items
5. Next best fixes

## Time rule

Do not spend the session repeating plans.

Review files, fix safe issues, commit, and continue.
