# UX Content Guide

## Purpose

Use this guide for marketing pages, campaign files, social posts, video ideas, Canva notes, and launch checklists.

The goal is to keep Wholelychit education content simple, useful, calm, and easy to act on.

## Main Rule

Help the visitor understand the next step quickly.

Use plain language. Avoid hype.

## Education Content Rules

For ReadEasy30 and MathEasy30 content:

- use short sentences
- use calm language
- avoid shame words
- avoid guaranteed-result claims
- respect adult learners
- keep parent/helper advice practical
- make the next step obvious
- connect reading and math only when the link helps the visitor

## Useful Page Pattern

A helpful education page should include:

1. Clear title.
2. Fast answer.
3. Who this helps.
4. Simple steps.
5. Common mistakes to avoid.
6. Helpful next action.
7. Link to the right practice app.

## Social Post Pattern

A useful post should include:

1. One learner problem.
2. One simple tip.
3. One calm next step.

Do not cram many ideas into one post.

## Short Video Pattern

A short video idea should include:

1. Hook.
2. One useful point.
3. One next step.

Keep the idea short enough for a 20 to 45 second video.

## Canva Asset Pattern

A Canva asset should use:

- large text
- calm colors
- simple icons
- one main message
- no clutter
- no tiny text

## Call To Action Rules

Good calls to action:

- Start reading practice.
- Start math practice.
- Try one calm step today.
- Save this checklist.
- Share with a parent, tutor, or helper.

Avoid pressure language.

## Repository Boundary

Website code belongs in each website repo.

Shared marketing, campaigns, social posts, video ideas, Canva notes, and content planning belong in `marketing-system`.

## AI Workflow

AI and Codex should update files directly when safe.

Do not ask Gerry to paste content into repo files when tools can do it.

If a large write is blocked, make a smaller safe commit and keep going.
