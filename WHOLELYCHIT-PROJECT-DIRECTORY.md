Use this file from any computer. Do not rely only on local paths such as `C:\Users\Carol\Documents\Codex\...`, because those exist on only one machine.

## Shared Operations Files

Use these shared files in `Wholelychit/marketing-system`:

- `WHOLELYCHIT-PROJECT-DIRECTORY.md` - project index
- `WHOLELYCHIT-SECURITY-PLAN.md` - security and risk rules
- `GOOGLE-AI-SEO-OPERATING-SYSTEM.md` - safe Google, YouTube, and AI search process
- `READEASY30-GOOGLE-AI-SEO-IMPLEMENTATION.md` - first-site SEO implementation model
- `SEO-SYSTEM.md` - reusable SEO page process
- `CONTENT-ENGINE.md` - content production process
- `MARKETING-CALENDAR.md` - publishing schedule
- `PROJECT-STATUS.md` - current project priorities

## Standard Repo Files

Each individual repo should eventually have:

- `AGENTS.md`
- `PROJECT-STATUS.md`
- `NEXT-STEPS.md`
- `SECURITY.md` for sensitive repos
- simple checks for links, sitemap, or critical app flow when useful

## Standard Rule

Do not add browser auto-click scripts to these repositories. Use clear repo instructions, checklists, and small commits instead.

## Working From Any Computer

On any computer:

1. Open GitHub.
2. Go to `Wholelychit/marketing-system`.
3. Open this file first.
4. Open `WHOLELYCHIT-SECURITY-PLAN.md` before sensitive work.
5. Use the correct individual repo for website code.
6. Keep shared plans in this marketing repo.
